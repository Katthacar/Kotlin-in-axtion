DSL construction
In this chapter, we’ll discuss how you can design expressive and idiomatic
API
s for
your Kotlin classes through the use of
domain-specific languages
(
DSL
s). We’ll explore
the differences between traditional and
DSL
-style
API
s, and you’ll see how
DSL
-style
API
s can be applied to a wide variety of practical problems in areas as diverse as
database access,
HTML
generation, testing, writing build scripts, defining Android
UI
layouts, and many others.
Kotlin
DSL
design relies on many language features, two of which we haven’t yet
fully explored. One of them you saw briefly in chapter 5: lambdas with receivers,
which let you create a
DSL
structure by changing the name-resolution rules in code
blocks. The other is new: the
invoke
convention, which enables more flexibility in
combining lambdas and property assignments in
DSL
code. We’ll study those fea-
tures in detail in this chapter.

Building domain-specific languages



Note that, outside of the
DSL
context, using an
object
as a parameter type rarely
makes sense, because it has only a single instance, and you can access that instance
rather than pass it as an argument. Here, it does make sense: the
object
is used not
to pass any data to the function, but as part of the grammar of the
DSL

start

should

StartWrapper

StartWrapper

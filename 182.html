Now let’s look at how you can convert values between different primitive types.
6.2.3
Number conversions
One important difference between Kotlin and Java is the way they handle numeric con-
versions. Kotlin doesn’t automatically convert numbers from one type to the other, even
when the other type is larger. For example, the following code won’t compile in Kotlin:
val i = 1
val l: Long = i
Instead, you need to apply the conversion explicitly:
val i = 1
val l: Long = i.toLong()
Conversion functions are defined for every primitive type (except
Boolean
):
toByte()
,
toShort()
,
toChar()
and so on. The functions support converting in
both directions: extending a smaller type to a larger one, like
Int.toLong()
, and
truncating a larger type to a smaller one, like
Long.toInt()
.
Kotlin makes the conversion explicit in order to avoid surprises, especially when
comparing boxed values. The
equals
method for two boxed values checks the box
type, not just the value stored in it. Thus, in Java,
new Integer(42).equals(new
Long(42))
returns
false
. If Kotlin supported implicit conversions, you could write
something like this:
val x = 1
val list = listOf(1L, 2L, 3L)
x in list
This would evaluate to
false
, contrary to everyone’s expectations. Thus the line
x in
list
from this example doesn’t compile. Kotlin requires you to convert the types
explicitly so that only values of the same type are compared:
>>> val x = 1
>>> println(x.toLong() in listOf(1L, 2L, 3L))
true
